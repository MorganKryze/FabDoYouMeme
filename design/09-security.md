# 09 — Security

## Checklist

| Concern                         | Mitigation                                                                                                                                                                                                                                                                                                                                                              |
| ------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Credential attacks**          | No passwords — magic link only; eliminates brute force, credential stuffing, and password storage breaches entirely                                                                                                                                                                                                                                                     |
| **Magic link prefetch**         | Verify endpoint is `POST`; email link routes to an intermediate frontend page (`/auth/verify?token=xxx`) that renders a "Log in" button. The button `POST`s the token — one extra click prevents pre-fetch bots from consuming the token                                                                                                                                |
| **Magic link replay**           | One-time use (`used_at` set on consume); 15-minute TTL                                                                                                                                                                                                                                                                                                                  |
| **Magic link leak**             | Token stored as SHA-256 hash only; raw token sent exclusively by email; hash is useless without the preimage                                                                                                                                                                                                                                                            |
| **Multiple active magic links** | On new magic link request, all existing unused tokens for that user are immediately invalidated (`UPDATE magic_link_tokens SET used_at = now() WHERE user_id = ? AND used_at IS NULL`). Only the latest link is ever valid                                                                                                                                              |
| **Email enumeration**           | Magic link endpoint always returns `200`, never reveals whether the email exists                                                                                                                                                                                                                                                                                        |
| **Silent email change**         | Notification sent to the old email address on any email change, regardless of who initiated it (user or admin)                                                                                                                                                                                                                                                          |
| **Session hijacking**           | `HttpOnly` + `Secure` + `SameSite=Strict` cookies; 30-day TTL renewed on each authenticated request                                                                                                                                                                                                                                                                     |
| **Stale admin/active status**   | `role` and `is_active` are re-fetched from the DB on every session lookup — never cached in the cookie or in memory. A deactivated or demoted user is blocked immediately                                                                                                                                                                                               |
| **CSRF**                        | `SameSite=Strict` cookie policy prevents cross-site form submission; origin check enforced on WebSocket handshake                                                                                                                                                                                                                                                       |
| **WS authentication**           | Session cookie validated during HTTP upgrade; unauthenticated upgrade requests rejected with `401` before WebSocket handshake                                                                                                                                                                                                                                           |
| **WS message flooding**         | Per-connection rate limit (20 msg/s); connections exceeding the limit are dropped                                                                                                                                                                                                                                                                                       |
| **WS payload bombs**            | `SetReadLimit(4096)` on every connection; frames exceeding 4 KB disconnect the client                                                                                                                                                                                                                                                                                   |
| **WS dead connections**         | Server sets 60s read deadline; reset on each received pong. Client sends ping every 25s. Connections that do not pong within 60s are cleaned up                                                                                                                                                                                                                         |
| **SQL injection**               | `sqlc` parameterised queries — no string concatenation in DB access code                                                                                                                                                                                                                                                                                                |
| **File upload abuse**           | Server validates declared `mime_type` against allowlist (JPEG, PNG, WebP) **and** validates magic bytes via `image.DecodeConfig` before issuing a pre-signed upload URL. Checking `Content-Type` header alone is insufficient — magic byte inspection cannot be bypassed by metadata manipulation                                                                       |
| **Upload size**                 | `size_bytes` validated server-side (`≤ MAX_UPLOAD_SIZE_BYTES`, default 2 MB) before issuing upload URL                                                                                                                                                                                                                                                                  |
| **Asset leakage**               | No public bucket; pre-signed download URLs embedded in `round_started` WS event (15-min TTL, `response-content-disposition=attachment`); session required for all access                                                                                                                                                                                                |
| **Brute force**                 | Per-IP rate limiting on all `/api/auth/*` endpoints (10 req/min)                                                                                                                                                                                                                                                                                                        |
| **Room creation spam**          | Rate limit: 10 rooms/hour per user                                                                                                                                                                                                                                                                                                                                      |
| **Upload URL spam**             | Rate limit: 50 upload URL requests/hour per admin                                                                                                                                                                                                                                                                                                                       |
| **Global API abuse**            | Rate limit: 100 GET requests/minute per IP across all endpoints                                                                                                                                                                                                                                                                                                         |
| **Invite enumeration**          | 12-char alphanumeric token (62^12 ≈ 475 trillion combinations); registration errors always return generic `400 {"code":"invalid_invite"}` regardless of whether the token exists, is expired, or is exhausted — no timing or status-code oracle                                                                                                                         |
| **Privilege escalation**        | Role checked server-side in middleware on every admin route; `is_active` checked on every session lookup; self-vote rejected server-side                                                                                                                                                                                                                                |
| **Secrets in repo**             | All secrets via `.env`; `.gitignore` includes `.env`; no signing keys required (raw random tokens, SHA-256 stored)                                                                                                                                                                                                                                                      |
| **SMTP in transit**             | `go-mail` enforces STARTTLS/TLS; plaintext SMTP disabled in production config                                                                                                                                                                                                                                                                                           |
| **Internal services**           | Postgres and backend not reachable outside Docker internal network; RustFS accessible only via reverse proxy externally                                                                                                                                                                                                                                                 |
| **Dependency drift**            | Go module checksums + `govulncheck ./...` in CI; `npm audit` on frontend dependencies                                                                                                                                                                                                                                                                                   |
| **CSP — frontend**              | Nonce-based CSP via SvelteKit `csp` option (`mode: 'nonce'`). Nonce generated per request in `src/hooks.server.ts`. Baseline policy: `default-src 'self'; script-src 'self' 'nonce-{n}'; style-src 'self' 'nonce-{n}'; font-src 'self'; img-src 'self' data: blob:; connect-src 'self' wss: ws:; frame-ancestors 'none'`. In production: replace `ws:` with `wss:` only |
| **CSP — API**                   | Go middleware sets `default-src 'none'` on all `/api/*` responses                                                                                                                                                                                                                                                                                                       |
| **XSS**                         | Svelte's default HTML escaping prevents reflected/stored XSS; CSP nonce provides defence-in-depth                                                                                                                                                                                                                                                                       |

## Threat Model Boundaries

This platform is **invite-only** and runs on **personal hardware**. The following threats are out of scope:

- **Public account enumeration**: no public registration or user listing
- **DDoS at infrastructure level**: assumed to be mitigated at the network layer or reverse proxy level; application-layer rate limits cover abuse by authenticated users only
- **Physical access to the host**: disk encryption and physical security are the operator's responsibility
- **Email interception**: the magic link email channel is assumed secure (TLS in transit); link compromise after delivery is mitigated by the 15-minute TTL and one-time use constraint

## Secrets Rotation

| Secret                                    | Rotation impact                             | Procedure                                                           |
| ----------------------------------------- | ------------------------------------------- | ------------------------------------------------------------------- |
| `POSTGRES_PASSWORD`                       | All backend connections drop                | Update `.env`, restart backend + postgres                           |
| `RUSTFS_ACCESS_KEY` / `RUSTFS_SECRET_KEY` | All asset operations fail                   | Rotate keys in RustFS admin console, update `.env`, restart backend |
| `SMTP_PASSWORD`                           | Email delivery fails (magic links broken)   | Update `.env`, restart backend                                      |
| Session tokens (in DB)                    | None — tokens are random and self-contained | N/A; invalidate individual sessions by deleting rows                |
